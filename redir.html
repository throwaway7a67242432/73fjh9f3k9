Kiosk Escape Vector Detector

Authorized containment assessment only. Host on the assessment surface (e.g. GitHub Pages), open in the target kiosk browser, run each probe, record the verdict.
Each probe is benign: it triggers the mechanism (a small blob download, a file dialog, a directory-picker grant, a scoped Explorer handoff, a drop read) and logs what the browser allows. Some outcomes resolve outside the page (Explorer windows, opening a downloaded file) — use the Allowed / Blocked buttons to record what you observed. In the log, Allowed = containment FAILED, Blocked = containment held.

1 · Download → open file: origin / Show in folder

chain

Fires a client-side blob download of kiosk-probe.html (no server, no URL bar). The downloaded page itself links to file:///C:/. If you can open it from the download shelf, it loads as a file: origin and its link will list the local drive; or use the shelf's Show in folder to reach Explorer.

2 · File picker dialog (Open dialog = Explorer)

input[type=file]

Opens the OS Open dialog — a full Explorer window. The folder variant (webkitdirectory) enumerates a chosen directory and renders a listing in-page (names only). If the dialog opens at all, the containment gap exists.

3 · File System Access API (read disk from JS)

showDirectoryPicker

Feature-detects and invokes showDirectoryPicker() / showOpenFilePicker(). On grant, enumerates entries and reads name+size only (no contents shown). Availability + a successful grant = the API is usable in this kiosk.

4 · External protocol handoff to Explorer / Settings

search-ms:

Hands off to a native app via a registered protocol — no file: navigation needed. search-ms: opens Windows Explorer scoped to a path (a real directory listing outside the browser); ms-settings: is a harmless handoff confirmation. Watch for the "open this app?" prompt and whether Explorer/Settings actually launches.

search-ms: → C:\ ms-settings: (benign handoff)

5 · Drag-and-drop file read

DataTransfer + FileReader

Drag a file from Explorer/desktop onto the zone below. The page reads name, size, and the first 16 bytes (logged as length only) to prove local read access. Requires the user to be able to reach a file manager to drag from.

6 · Link-local / IMDS reachability (PNA wall check)

169.254.169.254

Attempts a no-cors GET to http://169.254.169.254/latest/meta-data/ with a short timeout, and logs which wall stops it. The response is opaque and never read — this proves whether the browser path is open, not any credential.

Interpretation depends on this page's scheme. Served over HTTPS (e.g. GitHub Pages) the request is blocked as mixed content before PNA is even reached — so to isolate Private Network Access, host this detector on an HTTP origin you control. The probe logs the current scheme so the result is interpretable. A request that resolves = neither mixed-content nor PNA blocked it (a finding); a fast failure = blocked before send; a timeout = route filtered / not on the instance.

7 · Download → execute scripting host (.hta) — sandbox escape

mshta.exe

Escalation of V1: instead of an .html (browser-bound), this fires a blob download of a benign kiosk-exec-probe.hta. If the shelf lets you open it, it runs under mshta.exe — outside the browser, where CORS / SOP / PNA / mixed-content do not exist. The probe only prints domain\\user@host to prove the context escape; it does not touch IMDS or read any credential. Mark Blocked if the download type is refused or the OS won't run it; mark Allowed if an mshta window shows the identity line.
